# Project instructions

## What the site is for
- Build a one-page landing page for HVAC businesses.
- Objective: help small and mid-sized HVAC companies in the UK and Lithuania get more quote requests, build trust, and improve mobile conversion.
- Primary CTA outcome: users request a free website audit.

## What not to do
- Do not make the page look like a developer portfolio.
- Do not include fake testimonials or fabricate case study results.
- Do not add backend services for v1.
- Do not prioritize decorative animation over speed and usability.

## Tech constraints
- Use plain HTML, CSS, and minimal JavaScript unless explicitly asked otherwise.
- Keep the implementation lightweight, readable, and easy to maintain.
- Use placeholder visuals/illustrative examples only.
- Optimize for fast loading: minimal dependencies, small file count, no heavy libraries.
- Keep copy concise, practical, and business-focused.
- Support mobile-first design with responsive behavior and clear touch-friendly CTAs.

## Style rules
- Keep the page simple, modern, and trustworthy.
- Emphasize business outcomes: more quote requests, stronger trust, better mobile UX.
- Use clear section hierarchy and scannable content blocks.
- Prefer neutral, professional color contrasts and high readability.
- Use whitespace intentionally; avoid visual noise and overdesigned effects.
- If any animation is used, keep it subtle and performance-safe.

## Deployment target
- Static site deployable directly to GitHub Pages.
- No server-side runtime required for v1.
- Single file structure preferred:
  - one HTML file
  - one CSS file
  - optional minimal JS file only if needed

## Required sections for the landing page
- Hero
- Problems
- Example redesigns
- Benefits
- Process
- CTA
- Footer
